# v4.8.0 — 2026-07-13

## Added

- **Deck Import & Analysis**: New `/decks/import` page for importing existing deck lists from Moxfield, Archidekt, TappedOut, EDHREC, and native export formats. Imported decks are analyzed for role coverage, CMC curve, theme fit, and duplicate cards.
  - Per-card duplicate resolution with lazy-loaded replacement pools (tiered by type + functional role), radio-select with HTMX swap
  - Fill suggestions for under-100 decks: 3 independent sections (role shortfalls, theme fit, general synergy), each scored by composite fitness — theme match (+3 pts), shortfall role (+2.5 pts), other staple role (+1 pt), EDHRec popularity; cards sorted by score descending within each section; teal role pills highlight shortfall roles, muted pills show other covered roles
  - Theme autocomplete on the import form with random placeholder examples (3 themes per page load, at most one Kindred/Tribal)
  - Kindred auto-detection: injects the matching catalog `X Kindred`/`X Tribal` theme when ≥15 non-land creatures share a subtype
  - Post-analysis theme correction via **Edit themes** toggle — pre-fills current themes, supports autocomplete, re-runs detection
  - Creature subtype breakdown in the Cards by Type panel (subtypes with ≥5 creatures shown as inline pills)
  - Unified collapsible theme list with **Manual** / **Commander** badges, card count, and top-10 cards by EDHRec rank
- **Tagging fix — counter type tag names**: `COUNTER_TYPES` in `tag_constants.py` changed from regex raw strings to plain strings; `re.escape()` now applied at search time so tag names display as `+0/+1` instead of `\+0/\+1`
- **Theme catalog & parquet cleanup**: Backslash-escaped counter tags (`\+0/\+1 Counters` etc.) removed from `config/themes/theme_catalog.csv` and `card_files/processed/all_cards.parquet`; both loaders now strip leading `\` as a safety net

## Fixed

- **Theme catalog escape characters**: `\+1/\+0 Counters`, `\+0/\+1 Counters`, `\+2/\+2 Counters` were displaying with literal backslashes in autocomplete and theme panels; root cause was regex raw strings used as tag names in `COUNTER_TYPES`
